Alembic Database Migrations for olmo-eval-internal
==================================================

This directory contains database migration scripts managed by Alembic.

## Quick Start

### Running Migrations

```bash
# Upgrade to latest version
alembic upgrade head

# Upgrade by one version
alembic upgrade +1

# Downgrade by one version
alembic downgrade -1

# Show current version
alembic current

# Show migration history
alembic history --verbose
```

### Creating Migrations

```bash
# Autogenerate migration from model changes
alembic revision --autogenerate -m "Add new column to experiments"

# Create empty migration (for data migrations)
alembic revision -m "Migrate legacy data"
```

## Configuration

### Database URL

Set the database URL using environment variables (recommended):

```bash
export OLMO_EVAL_DB_URL="postgresql+psycopg://user:password@host:port/database"
```

Or set individual components using standard PostgreSQL environment variables (compatible with Beaker):

```bash
export PGHOST="localhost"
export PGPORT="5432"
export PGDATABASE="olmo_eval"
export PGUSER="postgres"
export PGPASSWORD="your_password"
```

Alternatively, edit `alembic.ini` and set `sqlalchemy.url`.

### For Production

Always review autogenerated migrations before applying them:

1. Generate migration: `alembic revision --autogenerate -m "description"`
2. Review the generated file in `alembic/versions/`
3. Test on a development database first
4. Apply to production: `alembic upgrade head`

## Migration Files

Migration files are stored in `alembic/versions/` with the format:
```
YYYYMMDD_HHMM_<revision_id>_<slug>.py
```

Each file contains:
- `upgrade()`: Forward migration
- `downgrade()`: Rollback migration

## Common Tasks

### Initial Setup (Fresh Database)

```bash
alembic upgrade head
```

### Adding a New Column

```bash
# 1. Modify the model in src/olmo_eval/storage/models.py
# 2. Generate migration
alembic revision --autogenerate -m "Add workspace column"
# 3. Review alembic/versions/<generated_file>.py
# 4. Apply migration
alembic upgrade head
```

### Rolling Back

```bash
# Rollback last migration
alembic downgrade -1

# Rollback to specific revision
alembic downgrade <revision_id>
```

## Integration with Code

The SQLAlchemy models are defined in:
- `src/olmo_eval/storage/models.py`

Alembic uses the `Base.metadata` from these models to autogenerate migrations.

## Best Practices

1. **Always use migrations in production** - Don't use `Base.metadata.create_all()`
2. **Review autogenerated migrations** - Alembic may miss some changes
3. **Test migrations** - Apply and rollback on development database first
4. **Keep migrations small** - One logical change per migration
5. **Don't edit applied migrations** - Create a new migration instead
6. **Backup before upgrading** - Always backup production database first

## Troubleshooting

### "Target database is not up to date"

```bash
alembic upgrade head
```

### "Can't locate revision identified by '<hash>'"

The database's migration state doesn't match the migration files. Reset:

```bash
# Check current version
alembic current

# If needed, stamp to a known version
alembic stamp head
```

### Schema already exists

If tables were created manually (e.g., via `initialize()`), stamp the database:

```bash
alembic stamp head
```

This tells Alembic the database is at the latest version without running migrations.
